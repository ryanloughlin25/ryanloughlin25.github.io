If statements in JavaScript and Ruby

1/13/2015

The if statement is more or less the same in any programming language I have ever used. Here is an example in Ruby: 


x = 15

if x.zero?
  puts "zero"
elsif x > 0
  puts "positive"
else
  puts "negative"
end

The code is simple, but it is still a good example of the structure of an if statement. There must be an if with a condition, there can be any number of elsifs, and there can be one else which has no condition. These 3 keywords make up the if statement in Ruby. Today I want to talk about elsif.

elsif is interesting because it is useful, but uncessary. The same code as above could be expressed as the following if we did not have elsif: 


x = 15

if x.zero?
  puts "zero"
else
  if x > 0
    puts "positive"
  else
    puts "negative"
  end
end

Here there are two if, each with an else, to accomplish the same thing as the first example with an if, an elsif, and an else. But the second example is harder to read and so it is good we have elsif.

I was surprised to learn that JavaScript does not have elsif, but handles the situtation such that elsif is not missed. Here are three ways to write the same example as above in JavaScript: 


//first
var x = 15;

if (x == 0)
  console.log("zero");
else if (x > 0)
  console.log("positive");
else
  console.log("negative");

//second
var x = 15;

if (x == 0)
  console.log("zero");
else {
  if (x > 0)
    console.log("positive");
  else
    console.log("negative");
}

//third
var x = 15;

if (x == 0)
  console.log("zero");
else
  if (x > 0)
    console.log("positive");
  else
    console.log("negative");

The first and second JavaScript examples are translations of the first and second Ruby examples. What's surprising is that the first example allows else if as two keywords and they simply work together. The third example shows why. You would think the third example would cause an error because you need braces around the second if statement. But you can have an else without braces if it only contains one statement and the second if statement is only one statement. So the only difference between the first and third example is whitespace.

There is no elsif keyword in JavaScript. JavaScript creates the illusion of elsif by simply letting you place else and if on the same line. It's a surprisingly elegant and simple approach.
