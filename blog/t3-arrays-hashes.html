Comparing basic usage of arrays and hashes

12/5/2014

Arrays and hashes can be tricky at first. Before we get into that though it's important to make sure we have a similar perspective on variables. For now, we can think of a variable as something with three properties - a name, a class, and a value. The following code (in irb) creates 2 variables. 

          
            > my_integer = 5
            #=> 5
            > my_string = "bananas"
            #=> "bananas"
The first variable is named my_integer, has a class of Fixnum (i.e. number), and has a value of 5. The second variable is named my_string, has a class of String, and has a value of "bananas". Ruby will use the value of a variable whenever we write the name of a variable. Lets see an example. 
          
            > 5 + 7
            #=> 12
            > my_integer + 7
            #=> 12
Both the code 5 + 7 and my_integer + 7 have a result of 12 because my_integer has a value of 5. Of course we could change the value of a variable later (we call them variables because the value varies) and then the result of any new code we write would use the new value.

Array is a class of variable. One way to make an array is like this. 

          
            > my_array = [5, 10, 15, 20]
            #=> [5, 10, 15, 20]
Now we have a variable named my_array with a class of Array and the value [5, 10, 15, 20]. As you can see this "value" is really multiple values at once. That's the point of an array.We can use the variable name my_array to refer to all the values at once or we can use the [] operator to refer to one value at a time. That sounds harder than it is, so lets try it. 
          
            > my_array
            #=> [5, 10, 15, 20]
            > my_array[0]
            #=> 5
            > my_array[1]
            #=> 10
            > my_array[2]
            #=> 15
            > my_array[3]
            #=> 20

Writing my_array gives us the whole array. Writing my_array[0] gives us the zeroth value and writing my_array[1] gives us the first value (remember computers start counting from 0). The indexes of the array are 0, 1, 2, and 3 and the associated values are 5, 10, 15, and 20. But what if we don't want the value 5 to be associated with 0? Maybe we put those numbers in there because it's a score for each player in a game. With arrays that's too bad. Each value is associated with a number starting with 0.

That's where hashes come in. The code to make a hash looks different than the code to make an array, but after we make them the code to use them is pretty similar. So first let's make a hash and let's use the same information as our array to keep it simple. 

          
            > my_hash = {0 => 5, 1 => 10, 2 => 15, 3 => 20}
            #=> {0=>5, 1=>10, 2=>15, 3=>20}
Now we have a variable named my_hash with a class of Hash and the value {0=>5, 1=>10, 2=>15, 3=>20}. Hashes are intended for storing pairs of things. Each pair has a key and a value. {0=>5, 1=>10, 2=>15, 3=>20} means that the key 0 is associated with the value 5, the key 1 is associated with the value 10, and so on. I said we were going to put the same information in the hash as we put in the array. Watch this. 
          
            > my_hash[0]
            #=> 5
            > my_hash[1]
            #=> 10
            > my_hash[2]
            #=> 15
            > my_hash[3]
            #=> 20
Look familiar? We can use the [] operator to get values from a hash just like we did with an array. The great thing about hashes is that we can also change the keys. If 5, 10, 15, and 20 are the scores for Beth, John, Anthony, and Susan in some game, then we can use a hash to associate their names with their scores. Their names will be the keys and their scores will be the values. Let's make a new hash where the keys are the player's names. 
          
            > scores_hash = {"Beth" => 5, "John" => 10, "Anthony" => 15, "Susan" => 20}
            #=> {"Beth"=>5, "John"=>10, "Anthony"=>15, "Susan"=>20}
Now we can use the [] operator to access each player's score using their name, which we couldn't have done with an array. 
          
            > scores_hash["Beth"]
            #=> 5
            > scores_hash["John"]
            #=> 10
            > scores_hash["Anthony"]
            #=> 15
            > scores_hash["Susan"]
            #=> 20
